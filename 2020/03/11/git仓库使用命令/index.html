git仓库使用命令

Git仓库命令

1、git init 初始化当前目录为Git仓库

2、git init newrepo 指定某个目录（newrepo）为git仓库

初始化后，会在目录下面出现一个名为.git的目录。所有git需要的数据个资源都存放在这个目录中。

3、

`
$ git add *.c

$ git add README

$ git commit -m ‘初始化项目版本’
`

以上命令是将目录下以.c结尾和README文件提交到仓库中，若想一次提交目录下所有文件 $ git add . 。然后提交附上注释。

4、git clone 从现有的git仓库中拷贝项目

git clone <repo> 克隆项目到当前目录 Git clone <repo> <directory> 克隆项目到指定目录

例如克隆 github 上的项目：$ git clone git@github.com:schacon/simplegit.git
克隆完后，在当前目录下会生成一个 simplegit 目录：

`
$ cd simplegit

$ ls
`
上述操作将复制该项目的全部记录

5、git status 命令用于查看项目的当前状态

6、git diff 命令显示已写入缓存与已修改但尚未写入缓存的改动的区别。

7、要从git中移除某个文件，就必须从已跟踪文件清单中移除，然后提交。从已跟踪文件中移除

git rm <file>

8、如果删除之前修改过并且已经放到暂存区域的话，则必须要用强制删除选项

git rm -f <file>
####9、如果把文件从暂存区移除，但仍希望保留在当前工作目录中
git rm --cached <file>

10、Git mv 用于移动或重命名一个文件、目录、软连接。

11、Git log 命令列出历史提交记录。

###Git远程仓库

由于你的本地git仓库和gitHub仓库之间的传输是通过ssh加密的，所以我们需要配置验证信息：

1.首先需要检查你的电脑是否已经有 SSH key

$ cd ~/.ssh
$ ls

2.创建一个SSH key

$ ssh-keygen -t rsa -C “youremail@example.com”

后面的邮箱是你在github上注册的邮箱，之后会请求确认路径和输入密码。一路回车就好。成功的话会在~/下生成.ssh文件夹，进去，打开id_rsa.puh（公匙），复制里面的key。然后在github上注册就好。

3.git remote add origin(你起的仓库名可修改) branch_Name（为空时默认为master）url 关联远程仓库，关联之后可以用git remote -v 来检查是否关联成功。

一般情况下含有共同文件时需要执行 git merge origin/master –allow-unrelated-histories 之后解决一下冲突。

4.git push -u origin master(分支，默认为master) 提交至远程仓库

####提取远程仓库

Git有两个命令用来提取远程仓库的更新。

1.git fetch 从远程仓库下载新分支与数据
2.git merge 从远端仓库提取数据并尝试合并到当前分支。

该命令就是在执行git fetch之后紧接着执行git merge远程分支到你所在的任意分支。假设你配置好了一个远程仓库，并且你想要提取更新的数据，你可以首先执行 git fetch [alias] 告诉git去获取它有你没有的数据，然后你可以执行git merge [alias]/[branch] 以将服务器上的任何更新合并到你当前分支。
