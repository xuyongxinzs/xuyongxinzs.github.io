在Ubuntu上搭建gitlab服务器

首先安装必须的一些服务

sudo apt-get update
sudo apt-get install -y curl openssh-server ca-certificates
sudo apt-get install -y postfix

这里要使用左右键和回车键选择确定、取消，弹出列表选项的时候，选择 Internet Site。我当初不知道还以为是卡了呢。

接着信任 GitLab 的 GPG 公钥:

curl https://packages.gitlab.com/gpg.key 2> /dev/null | sudo apt-key add - &>/dev/null

配置镜像路径

由于国外的下载速度过慢，所以配置清华大学镜像的路径。

vi /etc/apt/sources.list.d/gitlab-ce.list

写入
根据你的版本，选择对于的内容写入/etc/apt/sources.list.d/gitlab-ce.list，详见Gitlab Community Edition 镜像使用帮助

deb https://mirrors.tuna.tsinghua.edu.cn/gitlab-ce/ubuntu xenial main

在这里我出现了问题。
我按照vi /etc/apt/sources.list.d/gitlab-ce.list这个代码想修改配置源文件时，出现了我无法操作的页面。如图：

现在有一行命令是后来有人指点才加上去的。本来是什么都没有的。并且怎么按页没反应。后来才知道vi是打开文本编译器的指令，在这个框里编辑文本也得也要固定的指令。Ubuntu vi 命令大全

当然你也可以直接打开文件在文件中直接复制粘贴进去，不用指令输入。按照指定路径，比如上面这个vi /etc/apt/sources.list.d/gitlab-ce.list
加完之后：

在gitlab-ce.list这个文档里粘贴上想配置的镜像路径。就好了

当然你会发现这个文档无法编辑，为只读状态。这时候你就百度一下如何以管理员的身份打开文件夹。就可以啊。
