gitlab搭建总结

gitlab搭建总结

继上两次搭建之后，终于终于是让我搭建成功了。其实早就应该成功了。但是却败给了我的无知。

之前的我一直纠结在哪里呢？
明明我都成功运行了，为什么就是访问不了。
根本原因在于我的ip地址没有设置正确。

对，没错。就是这个ip，就是所有教程以及官网都告诉我安装成功后只需要该变这个ip为本机ip在重新初始化一下gitlab就可以了的地方。我硬生生的给写成了个网络ip（就是那个你百度一下ip就出来的网络ip）。

附上gitlab的一些常用命令：
sudo gitlab-ctl stop停止
sudo gitlab-ctl start开启
sudo gitlab-ctl restart重启
sudo gitlab-ctl status查看状态
sudo gitlab-ctl reconfigure确认配置（修改配置后，必须执行）
sudo gitlab-ctl tail 查看日志
卸载
执行如下四步：
sudo gitlab-ctl uninstall
sudo dpkg -r gitlab-ce
sudo rpm -e gitlab-ce
reboot（重启）

参考文献：在ubuntu16上搭建gitlab（实测可用）
在Ubuntu上搭建Gitlab服务

本地ip和网络ip的区别

本地ip就是你自己电脑私有的ip，可以自己随意修改。
网络ip是指本地网络（局域网）接入互联网由ISP服务商分配的固定IP或动态IP。

参考文献：本地ip和网络ip
本地内网IP和外网IP的区别 网络IP地址图文介绍

修改gitlab管理员密码

进入终端管理员模式

  1. gitlab-rails console -e production
    1. user = User.where(id: 1).first或者user = User.find_by(email: ‘admin@example.com’)
    2. user.password = ‘secret_pass’
    3. user.password_confirmation = ‘secret_pass’
    4. user.save!（切记）
    5. quit

注意：密码没有使用引号，奇怪的是使用单引号或双引号，密码就无效，估计是包含了这个字符，不包含，就没有问题。
